## Start the server

First, start the pyMessenger server in a terminal window:

```bash
python3 server.py
```

The server starts on port 1315 by default. You should see:

```
[+] SSL/TLS encryption enabled
[i] Using certificate: /home/user/.secure_messenger/certs/server.crt
[+] Server listening on 0.0.0.0:1315
```

### Server options

You can customize the server address and port:

<CodeGroup>

```bash Default
python3 server.py
```

```bash Custom port
python3 server.py --port 8080
```

```bash Custom host and port
python3 server.py --host 192.168.1.100 --port 8080
```

</CodeGroup>

<Note>
  Keep the server running in this terminal. Open new terminals for client connections.
</Note>

## Start your first client

In a new terminal window, start the client:

```bash
python3 client.py
```

### Client options

Connect to a remote server or custom port:

<CodeGroup>

```bash Localhost (default)
python3 client.py
```

```bash Remote server
python3 client.py --host 192.168.1.100
```

```bash Custom port
python3 client.py --host 192.168.1.100 --port 8080
```

</CodeGroup>

## Create your account

On first run, you'll see the login screen. Create a new account:

<Steps>
  <Step title="Select register option">
    Enter `1` to register a new account.
  </Step>
  <Step title="Enter username">
    Choose a unique username (minimum 3 characters).
  </Step>
  <Step title="Set password">
    Create a strong password (minimum 6 characters).
    
    <Warning>
      Your password encrypts your private key locally. Choose a strong password and don't forget it - there is no recovery mechanism.
    </Warning>
  </Step>
  <Step title="Key generation">
    The client automatically generates your RSA-2048 keypair and encrypts it with your password using PBKDF2-SHA256.
    
    Your private key is saved to: `~/.secure_messenger_client/keys/{username}_private.key`
  </Step>
</Steps>

The registration process executes this code from `client.py:457-485`:

```python client.py
# Generate RSA keypair
print_formatted_text(ANSI(f"{CYAN}[i]{RESET} Generating encryption keys..."))
rsa_key = RSA.generate(2048)
pubkey_bytes = rsa_key.publickey().export_key()

# Save encrypted private key
self.key_manager.save_private_key(
    username, 
    rsa_key.export_key(), 
    password
)

# Register with server
send_json(self.client_socket, {
    'type': 'auth_request',
    'auth_type': 'register',
    'username': username,
    'password': password,
    'pubkey': base64.b64encode(pubkey_bytes).decode('utf-8')
})
```

## Send your first message

Once logged in, you're in broadcast mode. Any message you send goes to all online users:

```
You: Hello, world!
```

Press Enter to send. All connected users will receive your encrypted message.

### Message encryption

Each message is encrypted using the hybrid encryption scheme:

1. Generate random AES-256 key
2. Encrypt message with AES-256-EAX
3. Encrypt AES key for each recipient with their RSA-2048 public key
4. Send encrypted envelope to server for routing

From `client.py:1156-1188`:

```python client.py
# Generate random AES key
aes_key = get_random_bytes(32)

# Encrypt message content
cipher = AES.new(aes_key, AES.MODE_EAX)
nonce = cipher.nonce
ciphertext, tag = cipher.encrypt_and_digest(plaintext.encode('utf-8'))

# Encrypt AES key for each recipient
encrypted_keys = {}
for target_user in targets:
    recipient_pubkey = RSA.import_key(self.peer_keys[target_user])
    cipher_rsa = PKCS1_OAEP.new(recipient_pubkey)
    encrypted_aes_key = cipher_rsa.encrypt(aes_key)
    encrypted_keys[target_user] = base64.b64encode(encrypted_aes_key).decode('utf-8')

# Send encrypted message
send_json(self.client_socket, {
    'type': 'encrypted_send',
    'targets': targets,
    'ciphertext': base64.b64encode(ciphertext).decode('utf-8'),
    'nonce': base64.b64encode(nonce).decode('utf-8'),
    'tag': base64.b64encode(tag).decode('utf-8'),
    'keys': encrypted_keys
})
```

## Use private messaging

### Enter a private room

Create a private chat room with another user:

```
/room alice
```

You'll see:

```
✓ Entering private room with: alice
════════════════════════════════════════
PRIVATE ROOM: You ⇄ alice
════════════════════════════════════════
```

All messages now go only to alice with the same end-to-end encryption.

### Leave the room

Return to broadcast mode:

```
/leave
```

### Send a single private message

Send a one-off message without entering a room:

```
/msg bob Hey, check out this file I'm sending!
```

## Share files securely

### Send a file

Send an encrypted file to another user:

```
/sendfile alice ~/Documents/report.pdf
```

The file is encrypted in 64KB chunks, each with its own AES-256 key. Progress is shown in real-time:

```
✓ File offer sent to alice
  Waiting for acceptance...

✓ alice accepted the file
[████████████████████████] 100% (2.5 MB / 2.5 MB)
✓ File sent successfully
```

### Receive a file

When someone sends you a file, you'll see:

```
📎 File offer from bob: presentation.pdf (1.2 MB)
   Use '/acceptfile' to accept or '/rejectfile' to reject
```

Accept the file:

```
/acceptfile
```

Or if you have multiple offers:

```
Pending file offers:
  1. presentation.pdf (1.2 MB) from bob
  2. spreadsheet.xlsx (0.8 MB) from alice

/acceptfile 1
```

Accepted files are saved to: `~/.secure_messenger_client/files/received/`

<Note>
  Files are prefixed with the sender's username to prevent conflicts: `bob_presentation.pdf`
</Note>

## Security considerations

### Challenge-response authentication

After initial registration, your password is never sent over the network. The login flow uses HMAC-based challenge-response:

From `client.py:545-566`:

```python client.py
# Receive challenge
challenge_msg = recv_json(self.client_socket)
nonce_b64 = challenge_msg.get('nonce')
salt_b64 = challenge_msg.get('salt')

nonce = base64.b64decode(nonce_b64)
salt = base64.b64decode(salt_b64)

# Derive password key
password_key = PBKDF2(
    password.encode('utf-8'),
    salt,
    32,
    count=100000,
    hmac_hash_module=SHA256
)

# Compute HMAC response
import hmac
response = hmac.new(password_key, nonce, 'sha256').digest()
```

### Rate limiting

<Warning>
  After 5 failed login attempts, your account is locked for 15 minutes to prevent brute force attacks.
</Warning>

### Session management

Authenticated sessions are valid for 24 hours. After this period, you'll need to log in again.
